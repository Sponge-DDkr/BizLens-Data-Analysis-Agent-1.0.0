## 图表类型速查

> 以下为 BizLens Visualization Agent 支持的图表类型，测试时用此表对照预期输出。

| chart_type | Plotly 实现 | 典型场景 | 预期 Trace 数 |
|-----------|------------|---------|-------------|
| 垂直柱状图 | `go.Bar()` | 类别对比 | 1 或 N（分组） |
| 水平柱状图 | `go.Bar(orientation='h')` | 排名对比 | 1 或 N |
| 分组柱状图 | `go.Bar()` × N, barmode='group' | 多系列类别对比 | N ≥ 2 |
| 堆叠柱状图 | `go.Bar()` × N, barmode='stack' | 百分比构成 | N ≥ 2 |
| 折线图 | `go.Scatter(mode='lines+markers')` | 时序趋势 | 1 或 N |
| 双轴组合图 | `make_subplots(specs=[[{"secondary_y":True}]])` | 柱状（左轴）+ 折线（右轴） | ≥ 2 |
| 饼图 | `go.Pie()` | 占比/份额 | 1 |
| 散点图 | `go.Scatter(mode='markers')` | 双变量关系 | 1 或 N |
| 多面板图 | `make_subplots(rows=N, cols=1)` | 多渠道/多维度并行趋势 | N × 每面板 trace 数 |

## 问题设计约定

每道题走完整 BizLens Agent 链路：

```
Planner（拆步骤）→ Code Interpreter（写代码+执行）→ Visualization（自动选图表类型）→ Insight（综合数据+知识库生成报告）
```

所有问题必须产出**至少一张图表**，L3 问题额外产出**带来源标注的业务归因**。

每条问题的格式为：
- **用户输入**：自然语言
- **图表要求**：明确需要的图表类型、方向、trace 数
- **Agent 链路**：系统内部执行路径
- **图表预期**：具体的图表规格
- **报告预期**：报告应包含的关键结论要点

## 一、sales_q3_2024 — SaaS 产品销售分析

### Q1.1 · 产品线营收对比 [L2]

**用户输入**：「Q3 各产品线的营收金额和订单数对比，用一张柱状图展示。」

**图表要求**：📊 双轴组合图——柱状图（左轴）= 营收金额，折线（右轴）= 订单数，X 轴 = 产品线（基础版/专业版/企业版），共 3 组

**Agent 链路**：

| 步骤 | Agent | 做什么 |
|------|-------|--------|
| 1 | Planner | 拆解：按产品线分组 → 汇总营收+订单数 → 对比排序 |
| 2 | Code Interpreter | `df.groupby('产品线').agg({'营收金额':'sum','订单数':'sum'})` |
| 3 | Visualization | 双轴组合图：柱状 = 营收金额（左轴）+ 折线 = 订单数（右轴） |
| 4 | Insight | 总结排名，5 段报告含「🎯 总结与建议」 |

**图表预期**：双轴柱线组合图（2 traces：Bar(营收金额, 左轴) + Scatter(订单数, 右轴)，产品线为 X 轴）

**报告预期关键点**：
- 数据事实：三产品线营收排名、订单数排名
- 总结与建议：直接回答哪个产品线最强，≤ 150 字

### Q1.2 · 区域营收 + 新老客占比 [L2]

**用户输入**：「四个大区的营收排名（用水平柱状图），再加一张新老客占比的百分比堆叠柱状图。」

**图表要求**：📊 两张图 — 左：水平柱状图（Y=区域，X=营收金额，4 条）；右：百分比堆叠柱状图（X=区域，每区域 2 段=新客/老客占比）

**Agent 链路**：

| 步骤 | Agent | 做什么 |
|------|-------|--------|
| 1 | Planner | 拆两步：区域营收排名 + 各区域新老客占比 |
| 2 | Code Interpreter | 步① `groupby('区域')['营收金额'].sum().sort_values()`；步② `groupby(['区域','是否新客'])['营收金额'].sum().unstack()` 算占比 |
| 3 | Visualization | 左图：水平柱状图 `go.Bar(orientation='h')`；右图：百分比堆叠柱状图 `barmode='stack'` |
| 4 | Insight | 「华东营收第一但新客占比最低——老客续费是主力；西南新客占比高，正在拓荒期」 |

**图表预期**：📊 水平柱状图（1 trace，4 条，营收降序）+ 📊 百分比堆叠柱状图（2 traces：新客/老客，4 组）

## 二、channel_roi_2024h1 — 渠道投放 ROI 分析

### Q2.1 · 渠道 ROI 排名 + 曝光转化散点 [L2]

**图表预期**：📊 水平柱状图（1 trace，6 条，ROI 降序）+ 🟢 散点图（6 traces，每渠道一个颜色，X=曝光量, Y=转化率）

### Q4.3 · 盈亏平衡预测 [L2]

**用户输入**：「画出月度营收趋势折线图，按近三个月平均增速外推未来 6 个月的预测营收（虚线），再画一条月均支出参考线，找出盈亏平衡月。」

> 📌 **优化说明**（2026-07-17）：原问题「保持当前增速 + 实际预测折线 + 置信区间阴影 + 盈亏平衡线」4 个任务堆在一句话里，置信区间阴影需要 `fill='tonexty'` 多 trace 组合，是图表 JSON 生成失败的头号原因。优化后：
> - ❌ 去掉了置信区间阴影（最易出错的 Plotly 高级特性）
> - ✅ 明确了外推方式：「最近 3 个月平均环比增长率 × 未来 6 个月」
> - ✅ 明确了盈亏平衡定义：「预测营收线」与「月度平均支出水平线」的交叉点
> - ✅ 指令有先后顺序（先...再...），Planner 更容易拆出正确步骤

**图表要求**：📈 折线图——实际月度营收（实线，go.Scatter）+ 预测营收（虚线，go.Scatter，line_dash='dash'）+ 月度平均支出参考线（水平虚线，add_hline），3 个 trace

**Agent 链路**：

| 步骤 | Agent | 做什么 |
|------|-------|--------|
| 1 | Planner | ①按月份汇总实际营收 ②计算近3月平均环比增长率 ③基于增长率外推未来6个月预测值 ④计算月均支出（金额<0 的月均值绝对值） |
| 2 | Code Interpreter | `groupby(月份)['金额'].sum()` 分收入/支出，算环比、外推序列、月均支出 |
| 3 | Visualization | 折线图：实际营收(实线) + 预测(虚线) + 月均支出 hline，3 traces |
| 4 | Insight | 「月均环比增长 X%，按当前趋势预计第 Y 个月实现盈亏平衡」 |

**图表预期**：📈 双折线 + 水平参考线（3 traces：实际营收实线 + 预测营收虚线 + 月均支出 hline），含交叉点标注
